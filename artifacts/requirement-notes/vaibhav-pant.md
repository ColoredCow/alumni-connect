 			     **ALUMNI CONNECT**  
	  
**What this platform solves:**   
Provides a shared space for students and alumni to interact, connect, share resources, guidance, and improve their job presence and networks.

**What this platform provides:**

* **For Students:**  
  1. Ability to interact with Alumni, seek mentorship and guidance.  
  2. Enhance their networking and seek internship/job opportunities.  
  3. Provides a common place to look for upcoming/ongoing events like summits, hackathons, open internship opportunities, etc.

	

* **For Alumni**  
  1. Allows alumni to recruit new graduates for their companies.  
  2. Allows them to recruit interested students for their own businesses/startups.  
  3. Allows them to promote their startup and showcase their business (if any).  
  4. Allows them to interact with other alumni.

   
**User verification:**  
A user can be:

1. An admin/moderator  
2. An alumnus.  
3. A student.

**Login system:**  
	The site can have a common login/registration page for any user.

Each user must fill all the fields including:

1. A dropdown that allows them to select whether they wish to login as a student, or an alumni (no separate field for admin as that would be assigned later such that a student or an alumni can be an admin)  
2. Full Name  
3. D.O.B.  
4. Branch  
5. Course Session  
6. Current year/semester (only for students)

   

Once logged in the user can only access some features such as viewing events,           
alumni list, but cannot appear in the list (if an alumni) or message anyone.  
In order to be fully verified the user must complete the verification process.  
	  
**Verification process:**

1. **For alumni:**   
1. Must upload a picture of their degree or enter their entry number (a number that uniquely identifies an admission)  
2. Once uploaded the request can appear in a common admin dashboard (in the backend) that the admin can choose to approve or reject.  
3. Once verified successfully an alumni can appear on the dashboard list with their details and accept other features such as chats, etc.  
4. The alumni can also choose to link their linkedin profile and the site can automatically fill in the details it fetched from their profile.

     **2\. For students:**

1.  Must upload a picture of their id card.  
2. Once uploaded the request can appear in a common admin dashboard (in the backend) that the admin can choose to approve or reject.  
3. Once verified successfully a student has access to features such as chats, viewing events, etc.

   

   **3\. For admins:**  
A UUID could be assigned to any fully registered user that could be used at the backend to make a user an admin.

After a record has been added it can be stored at a separate database.

**Features:** 

1. **Events:**  
   1. Events are special activities that could be posted by an alumnus, or an admin that students can view  
   2. An event section could be added to the side panel that would allow access to this feature.  
   3. Examples of events include summits, conferences, hackathons, internships openings, etc.

     ** 2\.  Chats:**  
		A small feature that allows a user to talk to another user directly.

1. Chats could be used to directly message an alumni or an admin (a setting could be added that a user can configure that allows them to accept direct messages).

      **3\. Alumni list:** 

1. A list of alumni with their full details.  
2. Helps students discover alumni.  
3. Alumni can be filtered and searched based on location, job position, company, branch, etc.

       **4\. Job/Internship openings:**

1. A panel that displays all the open job listings, internship opportunities   
2. These could be added by an alumni (and can optionally be first verified by an admin before it appears).  
3. Additionally startups can also post open positions for the business here.
