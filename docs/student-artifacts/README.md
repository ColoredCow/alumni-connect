# Student Artifacts

Requirement docs the first-year (CSE) students wrote and submitted during the foundation phase.

## Why this folder exists

Two reasons:

1. **So their early work isn't lost.** These docs were the first real thinking done on the project. Even where they're rough or get superseded by decisions later, they're part of the project's record.
2. **To help later joiners onboard.** If you're new and wondering what a first contribution looks like, this is what people before you did. Read a couple — they show how the problem was understood at the start.

## What goes here

- Requirement docs submitted by students
- Other early artifacts worth keeping (notes, sketches, problem write-ups)

## What doesn't

These are a snapshot, not living documents. Decisions and current thinking live in the main `docs/` files ([architecture.md](../architecture.md), [research.md](../research.md), etc.) — don't update artifacts here to match later decisions; that's the point of keeping them.

## Adding a doc

Name it descriptively (e.g. `requirements-data-collection-<name>.md`), drop it in this folder via a PR.
